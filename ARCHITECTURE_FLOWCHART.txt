                            🔄 FLUX DE DONNÉES

1. INPUT PHASE
   └─► Utilisateur lance streamlit run zedicus_v3.py
       └─► Streamlit initialise la page
           └─► TechnicalAnalyzer.fetch_data() via yfinance

2. ANALYSIS PHASE
   └─► Parallel Analysis:
       ├─► TechnicalAnalyzer:
       │   └─► identify_candle_patterns()
       │   └─► calculate_support_resistance()
       │   └─► calculate_indicators()
       │
       └─► NewsAnalyzer:
           └─► fetch_bce_news()
           └─► analyze_sentiment()
           └─► classify_news_impact()

3. DECISION PHASE
   └─► TradingSignalGenerator:
       └─► generate_entry_points()
       └─► generate_exit_points()
       └─► generate_trade_signal() [VOTING]
           └─► Composite score from all modules
           └─► Final action: BUY/SELL/WAIT

4. CACHING PHASE
   └─► Cache results for fast retrieval
       └─► Memory cache (TTL: 1h)
       └─► Disk cache (Persistent)

5. OUTPUT PHASE
   └─► Streamlit renders:
       ├─► Metrics
       ├─► Charts
       ├─► Recommendations
       └─► News impact
